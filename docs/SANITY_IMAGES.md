# Sanity images: URL builder + LQIP

## Image URL builder (`@sanity/image-url`)

The app uses `imageUrlBuilder` from `@sanity/image-url` with the Sanity client. See `src/lib/image.ts`:

- **`urlFor(source)`** — returns a builder (or `null` if no asset). Chain methods then call `.url()`.
- **`buildImageUrl(source, options)`** — convenience for width/height/fit/quality.

**Common usage:**

```ts
import { urlFor } from '@/lib/image';

// Responsive: width, height, fit
urlFor(image)?.width(800).height(600).fit('max').url();

// Quality (e.g. thumbnails)
urlFor(image)?.width(400).quality(60).url();

// Hotspot/crop are applied automatically from the image object (hotspot/crop fields from Sanity).
```

**Builder methods:** `.width()`, `.height()`, `.fit('max'|'min'|'fill')`, `.quality(1–100)`, `.format('webp'|'jpg'|...)`, `.auto('format')`, `.url()`.

## LQIP (Low-Quality Image Placeholder)

Sanity can store a small base64 blur on the asset **metadata** (`metadata.lqip`). Use it for blur-up placeholders with `next/image`:

1. **Request LQIP in GROQ** when you need a placeholder (optional; only if you want blur-up):

   ```groq
   "coverImage": {
     ...coverImage,
     "asset": {
       ...coverImage.asset,
       "metadata": { "lqip": coverImage.asset->metadata.lqip }
     }
   }
   ```

   Or at the asset level: `"asset": asset->{ ..., "metadata": { "lqip": metadata.lqip } }`.

2. **Use in Next.js:** pass `placeholder="blur"` and `blurDataURL={image.asset?.metadata?.lqip ?? undefined}` to `next/image`. If you don’t request `lqip`, use a static small data URL or skip blur placeholder.

**Note:** LQIP is generated by Sanity when the image is processed. No extra config required; request it in the query when needed.
